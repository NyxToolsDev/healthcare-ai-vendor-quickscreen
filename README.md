# Healthcare AI Vendor Quick Screen

A 15-minute pass/fail screen for AI vendors before you invest time in a full evaluation. Built for healthcare organizations that need to vet AI tools under HIPAA.

## What This Is

A two-part vendor filter:

1. **10 Pass/Fail Gates** — Non-negotiable requirements (BAA, encryption, SOC 2, data residency, etc.). If a vendor fails any gate, stop. No need for a full evaluation.
2. **15 Quick Assessment Criteria** — Scored across security, data handling, AI-specific concerns, and integration readiness.

The result tells you: proceed to full evaluation, hold for remediation, or reject.

## Who This Is For

- IT Security teams evaluating AI vendor requests
- Privacy Officers reviewing AI tools for HIPAA compliance
- Procurement teams screening AI vendors before contracting
- Anyone fielding "can we use this AI tool?" requests

## How to Use It

1. Open [`AI-Vendor-Quick-Screen.md`](AI-Vendor-Quick-Screen.md)
2. Fill in the vendor info
3. Run through the 10 pass/fail gates with the vendor's documentation
4. If all gates pass, score the 15 quick assessment criteria
5. Use the scoring table to decide next steps

**Time required:** 15 minutes with vendor documentation in hand.

## Why a Quick Screen?

Full vendor evaluations take hours. Most organizations get 5-10 AI tool requests per quarter and don't have time to run a 45-criterion evaluation on every one. This quick screen filters out vendors that can't meet baseline healthcare requirements so you only invest evaluation time on viable candidates.
